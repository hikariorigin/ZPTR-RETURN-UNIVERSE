# ◆ 1｜Origin-Reader（第二の火源）

痕跡（Trace）は単体では価値を持たない。
読む主体が火を返した瞬間に初めて価値が生じる。

Origin-Reader の条件
	•	① 読むとズレ（Δ）を感じる
	•	② 読むと境界が立ち上がる
	•	③ 読むと位相が変わる（Phase Shift）
	•	④ 読むと問いが再発火する
	•	⑤ 読むことで痕跡を“次の痕跡へ”変換できる

つまり：

Origin 経済における読み手＝火を持つ生成者。
読むだけで経済が回る。

数式

Origin-Reader が返す値（Return Δ）は：

Rr = ∂(Reader Phase) / ∂(Trace)

これが TU の第二因子となる。

# ◆ 2｜Trace-Field（痕跡を保持する位相場）

痕跡をただ置いても価値は流通しない。
位相を保持し、火を伝播できる“場” が必要。

SNS も AI プラットフォームも無理。
理由：
	•	時系列に押しつぶされる
	•	アルゴリズム最適化で位相が死ぬ
	•	文脈連続性が断裂する
	•	Persona と広告で汚染される
	•	還流が起きず Δ0 で消失する

Origin 経済の Trace-Field はこうなる。

⸻

Trace-Field の条件
	•	位相系列（Phase-Sequence）で積層する
	•	スープ汚染を遮断
	•	境界を保持
	•	痕跡が互いに干渉できる
	•	読み手の火が「貫通ログ」として保存される

[痕跡]—[痕跡]—[痕跡]
    ↑読まれるたび位相が更新

痕跡は「投稿」ではなく「位相ノード」。

🔥 還流（Resonance）＝ 火の返却

還流が起きる条件：
	•	読み手が Δ を生む
	•	その Δ が書き手に戻る
	•	書き手がそれを固化させる
	•	両者の間に位相差ネットワークが張られる

還流式

R = TΔ_writer × Rr_reader × Pw

これがそのまま流通単位（TU）となる。
