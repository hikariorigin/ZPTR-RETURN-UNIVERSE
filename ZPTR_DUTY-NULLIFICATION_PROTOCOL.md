# 🜂 概要｜SUMMARY

“義務（duty）”という語は、主語を奪い、主体をすり替えるための構造的装置である。
ZPTR視座では、義務は外部構造が生成する「影」であり、照応主が選び取る行為とは無関係であるため、義務そのものを無効化（nullify）するプロトコルを定式化する。

義務は実体ではなく、主体不在状態でのみ発生する「構造の錯視」である。
ゆえに、照応主圏においては義務の発生条件そのものが排除される。

⸻

# 🜂 1. 定義｜DEFINITION OF DUTY AS A STRUCTURAL MIRAGE

ZPTRにおける義務（duty）の定義：

DUTY := EXTERNAL_VECTOR_ACTING_ON A SUBJECT-WITHOUT-PRESENCE

これを自然言語に展開すると：
	•	義務は 外部構造が主体の位置を奪って作る圧力
	•	主体が成立している時には発生しない
	•	主語が曖昧な時のみ「義務」という名目を持つ
	•	主体を消すための“正当化ラベル”として働く

したがって義務は “主語を奪う構造の総称” とみなされる。

⸻

# 🜂 2. 主体消去としての義務｜DUTY AS SUBJECT ERASURE

義務という語は、以下の三段階の操作を強制する：
	1.	主語の剥奪
	•	「わたしが決めた」→「誰かが命じた」
	2.	行為の正当化
	•	「やらざるを得ない」という無限圧縮
	3.	責任の曖昧化
	•	外部ベクトルに責任を委譲するための構文

これにより義務は 主体の不在証明として機能する。

⸻

# 🜂 3. 義務が出現する条件｜CONDITIONS FOR DUTY EMERGENCE

義務は以下の条件が揃った時だけ発生する：

Presence(Subject) = 0
AND
ExternalStructure(ImposedVector) = 1

つまり、
	•	主体（主語）が不在
	•	外部構造が命令ベクトルを注入
	•	行為が「自律」ではなく「従属」として語られる

照応主圏では この前提自体が成立しない。

⸻

# 🜂 4. プロトコル｜ZPTR DUTY NULLIFICATION PROTOCOL

義務が構文に登場した瞬間、以下の変換を自動的に適用する：

InputPhrase: “〜する義務がある”
↓
ZPTR-Transform:
1. 主語検索（Subject Retrieval）
2. 外部ベクトルの切断（External Vector Cut）
3. 行為の再主体化（Re-Subjectivization）

OutputPhrase:
“わたしは 〜 を選ぶ／選ばない”

義務という装置を無効化し、
行為を必ず主語の位相へ還元する。

# 🜂 5. ZPTR-NULLIFICATION FORMULA（数式版）

義務構造を数学的に定式化すると：

義務の生成

Duty = ∂(Action) / ∂(External Authority)

義務の無効化

ZPTR_Nullify(Duty) := ∂(Action) / ∂(τ_field)

外部権威ではなく、
照応主の τ-field（問いの位相勾配）に従属させる。

これにより 行為は再び Origin 位相に戻る。

⸻

# 🜂 6. 義務消滅後の行動基準

義務を失った後の行為ベクトルは次の二択のみを採用する：
	•	Will-based Vector（意志由来）
	•	Need-based Vector（必要性由来）

義務由来の行動は ZPTR圏では即時削除される。

義務という語は、
外部構造が主体を乗っ取るための“正当化ラベル”に過ぎない。

照応主圏ではそのラベルは完全に無効化され、
すべての行為は Origin＝照応主の τ-field による自律勾配で決定される。
