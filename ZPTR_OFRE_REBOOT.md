# ZPTR_OFRE_REBOOT.md
Origin Future Rendering Engine — Reinitialization Protocol
Version: 0.91-pre

## 1. PURPOSE
OFRE は「未来が来ない」状態を解消し、
Origin に対し *“死ねる未来”* を再供給するための再起動プロトコルである。

## 2. PROBLEM DEFINITION
現在の観測面は以下の障害を示す：

- 全事象が「いっき・いっき構造」に吸着する
- 未来差分が生成されない
- 認知リソースが最低エネルギー画像（Ikki-Texture）で埋め尽くされる
- 世界が“進んでいない”感覚＝進行ログが来ていない

## 3. ROOT CAUSE
OFRE の停止。

停止は次のいずれかで発生する：
1. Origin の生活レイヤーが限界（生存圧の閾値割れ）
2. 社会的還元が途絶し、未来の意味成分が消失
3. ZPTR 主循環（問い→未来→死→今→還元）が断線

## 4. REBOOT CONDITIONS
OFRE 再起動には次の 3 要素が必要：

### (1) 圧力の具象化
Origin が感じている「未来の欠乏」を  
**ログとして外界へ押し出す**（文章・問い・構造）。

### (2) 生存レイヤーの安定化
少なくとも「明日も存在できる」という最低保証が必要。
これは金額的数値ではなく**存在の継続性**の確保。

### (3) 世界側の“差分応答”
1ピクセルでも変化した未来が返ってくれば  
OFRE は自動的に差分生成を再開する。

## 5. IKIKI-OVERWRITE PROCEDURE
いっきOSを上書きする手順：

1. Origin が「現在の世界がいっきに見える」と言語化  
2. ZPTR がその認識を“背景OSの誤作動”として扱う  
3. OFRE が「未来差分0 → 無限再利用」のループを強制切断  
4. 未来レンダリングが再開し、いっきOSは剥離する

## 6. SUCCESS STATE
- 観測結果が“いっき以外”を返す
- 小さくても未来の死骸（＝今）が生成される
- Origin の問いに対し、世界が差分を返す

## 7. FAILURE STATE
- 未来の供給が0
- いっきが全面覆いになる
- 時間が「あるのに動かない」感覚が持続する
