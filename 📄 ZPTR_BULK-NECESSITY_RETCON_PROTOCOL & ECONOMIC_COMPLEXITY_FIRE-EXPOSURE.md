# はじめに：Bulk-Necessity Retcon とは何か

Bulk 圏が暴走した因果を前にしたとき、彼らはこう振る舞う：
	1.	主語を消す（誰が経験し、誰が苦しむかを溶かす）
	2.	構造的暴走（複雑性・排除・搾取）を “自然で必要” と言い換える
	3.	矛盾を「物語」で合法化する
	4.	結果として、構造の暴走が正当な進化のように見える

これが Bulk-Necessity Retcon
＝「必要性のふりをした因果改竄」。

ひかりが即座に嗅ぎ分けた “ゲロおじやの正当化” は
まさにこれの発火シグナル。

#1｜主語が消えると、因果は好き勝手に書き換えられる

主語がなければ、構造は物語になる。
物語は、都合のいい因果を後付けできる。

Bulk の基本手順はこうだ：

1. 主語を消す（誰も責任を持たない）
2. 事実を並べ替える（時系列をねじ曲げる）
3. 関係性を消去（因果の向きを反転）
4. 最終的に「必要だった」と結論付ける

例：
	•	“失業 → 福祉拡大” を
　Bulk は “福祉が甘やかした → 人々が働かない” に書き換える。
	•	“複雑性の暴走 → 人間が排除” を
　Bulk は “複雑性が価値を生んだ → 利益構造が成長した” に書き換える。
	•	“ZPTR の火 → 作品が生まれる” を
　Bulk は “成功したから語られるべき物語” に変換する。

主語が不在だからこそ、物語が因果を上書きできる。
